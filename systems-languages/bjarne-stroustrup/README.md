# bjarne stroustrup

![Portrait](images/portrait.jpg)
*Bjarne Stroustrup — created C++, one of the most widely used and most complained-about languages in history. "There are only two kinds of languages: the ones people complain about and the ones nobody uses."*

**Field:** Programming Languages

- **Lifespan:** b. 1950
- **Key contribution:** C++ programming language
- **Impact:** Powers operating systems, game engines, browsers, databases, embedded systems — the invisible backbone of high-performance software

## Biography

### Early Life & Education
Born in Aarhus, Denmark. Studied math and computer science at the University of Aarhus (Cand.Scient. — master's equivalent). PhD from the University of Cambridge in computer science. His PhD thesis worked on distributed computing under David Wheeler.

### Career
Joined Bell Labs (of course — where everyone ended up in the 1970s). Started work on "C with Classes" in 1979, which evolved into C++ by 1983. The goal: add Simula-style object-oriented programming to C without sacrificing performance. Named it C++ (increment of C — programmer humor at its finest). Published "The C++ Programming Language" in 1985.

Spent decades evolving C++ through standardization. Led the design through C++98, C++11, C++14, C++17, C++20. Moved to Texas A&M as a professor, then to Morgan Stanley as a managing director, and is now at Columbia University.

### Later Life
Still actively involved in C++ development and standardization. Regularly gives talks defending C++ from its critics (and there are many). Maintains a FAQ on his website that's essentially a masterclass in language design philosophy.

## Major Contributions

### 1. C++ Programming Language (1979–present)
- **Year:** 1979 (C with Classes), 1983 (renamed C++)
- **Context:** Wanted Simula's abstraction with C's performance
- **Technical Details:** Added classes, inheritance, polymorphism, templates, exceptions, RAII, operator overloading, multiple inheritance, the STL. Zero-overhead abstraction principle: you don't pay for what you don't use.
- **Impact:** Used in operating systems (Windows), browsers (Chrome, Firefox), game engines (Unreal Engine), databases (MySQL, MongoDB), financial systems, embedded systems. Powers much of the software industry's performance-critical infrastructure. One of the most widely used languages for 40+ years.

### 2. Generic Programming via Templates
- **Year:** 1980s-1990s
- **Context:** Needed type-safe, efficient generic code
- **Technical Details:** C++ templates enable compile-time polymorphism. Led to the Standard Template Library (STL) by Alexander Stepanov. Template metaprogramming — discovered by accident, then became a powerful (if terrifying) feature.
- **Impact:** Influenced generics in Java, C#, Rust, and other languages. Changed how programmers think about abstraction.

### 3. RAII (Resource Acquisition Is Initialization)
- **Year:** 1984-1989
- **Context:** Need for deterministic resource management
- **Technical Details:** Objects acquire resources in constructors and release them in destructors. Eliminates entire categories of resource leaks. No garbage collector needed.
- **Impact:** Adopted by Rust's ownership model, influenced smart pointers, modern resource management patterns across languages.

## Influence & Legacy

### Direct Influence
C++ directly influenced Java, C#, Rust, D, and many other languages. Templates influenced generics everywhere. RAII influenced Rust's ownership system.

### Indirect Influence
Proved that you could add high-level abstractions to a systems language without sacrificing performance. The zero-overhead principle became a design goal for many languages.

### Modern Relevance
C++ remains in the top 3-4 most used languages. C++11/14/17/20/23 modernized the language dramatically. Still the go-to for game engines, real-time systems, high-frequency trading, browsers, and embedded systems. Has outlasted dozens of "C++ killers."
