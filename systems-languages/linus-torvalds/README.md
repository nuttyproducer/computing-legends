# linus torvalds

![Portrait](images/portrait.jpg)
*Linus Torvalds — created Linux because he wanted a free Unix for his PC, then created Git because he needed better version control for Linux. Both projects are now essential infrastructure for modern civilization. Also known for his... colorful email style.*

**Field:** Operating Systems, Version Control

- **Lifespan:** b. 1969
- **Key contribution:** Linux kernel, Git version control system
- **Impact:** Linux powers 96%+ of servers, all Android phones, most embedded systems, and the world's top supercomputers. Git is used by virtually every software developer on Earth.

## Biography

### Early Life & Education
Born in Helsinki, Finland. Named after Linus Pauling (the Nobel Prize chemist). Grew up in a Swedish-speaking Finnish family. Got hooked on programming with a Commodore VIC-20 at age 10. Studied computer science at the University of Helsinki.

### Career
In 1991, while a student at Helsinki, Torvalds bought a 386 PC and wasn't satisfied with the available operating systems. Minix (Andrew Tanenbaum's educational Unix clone) was limited, and real Unix was expensive. So he started writing his own Unix-like kernel — "just a hobby, won't be big and professional like GNU." That "hobby" became Linux.

Released Linux 0.01 on September 17, 1991. Made it open source under the GPL license. The kernel attracted contributors from around the world. Combined with GNU tools (from Richard Stallman's project), it became a complete operating system — technically "GNU/Linux" but everyone just says "Linux."

In 2005, created Git in about two weeks after the Linux kernel community lost access to the proprietary BitKeeper version control system. Git was designed for speed, distributed development, and handling the massive Linux kernel codebase. It became the standard version control system for software development worldwide.

Famous for his abrasive communication style on the Linux kernel mailing list (LKML). In 2018, took a break to work on his interpersonal skills and adopted a Code of Conduct for the kernel community. Returned notably more diplomatic — by his standards.

### Later Life
Lives in Portland, Oregon. Works for the Linux Foundation. Still personally manages Linux kernel releases and merges patches. Scuba diving enthusiast (wrote Subsurface, a dive log application). Uses a non-descriptive desktop setup — not the server rooms his kernel powers.

## Major Contributions

### 1. Linux Kernel (1991)
- **Year:** 1991
- **Context:** Student who wanted a free Unix-like OS for his new 386 PC
- **Technical Details:** Monolithic kernel (famously debated with Tanenbaum about this vs. microkernels). Written in C (and some assembly). Modular design, loadable kernel modules. Supports virtually every hardware architecture. Thousands of contributors — the largest collaborative software project in history.
- **Impact:** Runs 96%+ of the world's top servers, all Android devices, most embedded systems, all 500 of the world's top supercomputers, space stations, Tesla cars, smart TVs, routers. The Internet literally runs on Linux. Cloud computing (AWS, Azure, GCP) is primarily Linux.

### 2. Git Version Control (2005)
- **Year:** 2005
- **Context:** BitKeeper revoked free license for Linux kernel; needed a replacement FAST
- **Technical Details:** Distributed version control — every clone is a full repository. Content-addressable storage (SHA-1 hashing). Branching and merging are cheap. Designed for speed and data integrity. Written in C, Perl, shell scripts.
- **Impact:** Standard VCS for software development. GitHub, GitLab, Bitbucket — all built on Git. Over 100 million developers use Git. Changed how software is collaboratively developed.

### 3. Subsurface (2011)
- **Year:** 2011
- **Context:** Linus is an avid scuba diver and wanted better dive log software
- **Technical Details:** Cross-platform dive log application. Qt-based GUI.
- **Impact:** It's a dive log app. Not everything has to change the world — sometimes you just want to track your dives.

## Influence & Legacy

### Direct Influence
Linux directly spawned Android, ChromeOS, SteamOS, and thousands of distributions. Git spawned GitHub, which changed software collaboration forever.

### Indirect Influence
Proved that open-source, community-driven development could produce world-class software. The Linux development model influenced countless other open-source projects.

### Modern Relevance
Linux IS modern computing infrastructure. Cloud computing, AI training, smartphones, IoT, supercomputing — all Linux. Git is so universal that "version control" and "Git" are nearly synonyms for most developers. GitHub has 100+ million developers.
