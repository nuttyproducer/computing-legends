## Guidelines

### Writing Style
- Use clear, accessible language
- Provide context for technical terms
- Be objective and factual
- Include citations for claims

### Images
- Place all images in the person's `images/` subdirectory
- Use descriptive filenames (e.g., `portrait.jpg`, `turing-machine.jpg`)
- Always include attribution and license information
- Optimize images (portraits under 200KB)
- Use public domain or properly licensed images only

### Links
- Link to other people in the repository when mentioned
- Use relative paths: `[Name](../category/person-name/)`
- Verify all external links are active

### Sections
- Not all sections are required—use what's relevant
- Add custom sections if needed for unique contributions
- Keep consistent formatting across all person pages

### Maintenance
- Update the "Last Updated" date when making changes
- Add your GitHub username to Contributors
- Keep information current and accurate
